    Accessible design is good design - it benefits people who don't have disabilities as well as people who do. Accessibility is about removing barriers and providing the benefits of technology for everyone

    Why is it difficult to diagnose accessibility issues?

    When a site has an issue with core functionality, it is easy to see. Something doesn't behave or appear correctly, and it is often obvious how it can be addressed.

    For users not utilizing assistive technologies, accessibility issues are often invisible

    How do we make accessibility issues easier to identify?

    We need to increase awareness of the types of issues that can arise with websites unoptimized for assistive technologies.

    We can integrate tooling into the development lifecycle to make them  visible.

    Tooling

    Things Tooling Can Find

    • Contrast issues in text
    • Form labels not associated with inputs
    • Missing alt attributes on images
    • Missing page metadata
    • Main inside of landmarks
    • Heading Hierarchy issues
    • List issues
    • Duplicate IDs/Link Text

    Automated testing can't catch all accessibility issues, but it can catch a lot. Estimates say automated scanning can catch from 20 to 57% of common issues.

    Automated testing does not replace a manual review, but it can reduce the surface-area needed for review and helps to ensure a high-quality codebase.

    Things Tooling Can't Find (yet)

    • Poor alt, heading, and hyperlink text
    • Poor tag usage
    • Bad reading or tab order
    • Hidden content
    • Poor focus/active/hover states
    • Unclear instructions and error messages

    Tools of the Trade

    Developer Tooling

    • AXE VSCode Extension
    • Google Lighthouse
    • CSS Bookmarklet
    • WAVE (Web Accessibility Evaluation Tool)

    Linting

    • Linters
      • Axe-core
      • Lighthouse (node module)
      • JSX-ally for React
      • AccessLint.js (Github Actions has a template for this)
      • Pa11y (can be set up with dashboards/monitoring)
    • Can setup pre-commit, pre-push hooks with Husky
    • Can add to CI/CD Pipeline to prevent merges/fail builds

    Testing

    • Unit Testing with standard tools
      • Can test interaction/focus states
      • Aria-states
      • Axe has unit testing helper functions
    • Integration Testing with headless browsers like Selenium, Puppeteer
      • Checks page/level rules
      • Color contrast
      • Keyboard navigation
